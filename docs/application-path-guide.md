# Application Path Guide

Last checked: 2026-06-17. Use this guide to decide the first form, portal, or email. It is not legal advice.

## Path 1: Startup or SME AI Innovation

1. Read AI Factory access modes: https://www.eurohpc-ju.europa.eu/ai-factories/ai-factories-access-modes_en
2. Choose scale:
   - Playground: first experiments and entry-level tests.
   - Fast Lane: HPC-ready work up to 50,000 GPU hours.
   - Large Scale: more than 50,000 GPU hours and high-impact AI work.
3. Apply through the EuroHPC access portal: https://access.eurohpc-ju.europa.eu/
4. If eligibility or factory choice is unclear, use the AI Factories helpdesk: https://www.eurohpc-ju.europa.eu/ai-factories/helpdesk-ai-factories_en

## Path 2: Researcher, Public Body, or EU Collaborative Project

1. Read AI for Science and Collaborative EU Projects: https://www.eurohpc-ju.europa.eu/eurohpc-ju-call-proposals-ai-science-and-collaborative-eu-projects_en
2. Compare standard supercomputer calls: https://www.eurohpc-ju.europa.eu/supercomputers/supercomputers-access-calls_en
3. Use Benchmark or Development first if you need scaling evidence or code readiness.
4. Use Regular or Extreme Scale only when the project justifies larger resources and longer review cycles.

## Path 3: IT4LIA-Specific Route

1. Read the EuroHPC Italy page: https://www.eurohpc-ju.europa.eu/ai-factories/italy_en
2. Read IT4LIA package descriptions: https://it4lia-aifactory.eu/what-we-can-do-for-you/
3. Choose a starting hypothesis:
   - TRY: first-time user, workload test, scalability trial.
   - Medium: fine-tuning, dataset building, agentic AI prototype.
   - Large: pre-training or foundation-model adaptation.
4. Contact IT4LIA through the consultation flow or public contacts:
   - General: `info-it4lia@cineca.it`
   - Services: `support-it4lia@cineca.it`
   - Contact page: https://it4lia-aifactory.eu/contact-us/

## Path 4: Open-Source Team Without a Legal Entity

Public sources reviewed do not confirm a direct route for a loose open-source collective. Limes interpretation: identify a genuine eligible host first, such as a startup/SME, university lab, research organisation, public body, or funded consortium.

Before outreach, decide:

- Who is the applicant?
- Who is the PI or responsible contact?
- Which organisation accepts terms and account responsibility?
- Who controls data and publication decisions?
- Which public repositories, model card, benchmark results, and safety notes are ready?

## Practical Decision Tree

| Question | If yes | If no |
| --- | --- | --- |
| Are you a startup or SME doing AI innovation? | Check Playground/Fast Lane/Large Scale. | Check research/public/consortium routes. |
| Do you need only a small test or scaling proof? | Check Playground, Benchmark, or Development. | Continue. |
| Are you a scientific/public-sector user? | Check AI for Science, Benchmark, Development, Regular, or Extreme Scale. | Continue. |
| Is Italy/IT4LIA a natural service route? | Ask IT4LIA about TRY/Medium/Large fit. | Use EuroHPC portal/helpdesk. |
| Do you lack a legal entity? | Find eligible host before applying. | Prepare portal artifacts. |

## Before Applying

Do not send a vague "we need GPUs" message. Send a concrete package:

- applicant type and country
- legal entity or host partner
- project summary and public outputs
- route hypothesis
- compute estimate
- data/licence statement
- benchmark or prototype evidence
- model/system card draft
- safety and compliance notes
- public repo links
